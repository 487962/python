# 과제 1

> 리눅스 프로세스 관리와 관련된 4가지 핵심 명령어 `top`, `ps`, `jobs`, `kill`에 대해 조사한 내용을 정리한 문서입니다.


## 1. `top`

### 개요
`top`은 **Table Of Processes**의 줄임말로, CPU·메모리 사용량이 높은 프로세스 순으로 정렬하여 **실시간(동적)** 으로 보여주는 명령어입니다. 화면이 일정 주기(기본 3초)마다 갱신되며, 시스템 전반의 부하 상태를 한눈에 파악할 수 있습니다.

### 기본 문법
```bash
top [옵션]
```

### 화면 구성
| 영역 | 내용 |
|------|------|
| 상단 요약 | 현재 시간, uptime, 로그인 사용자 수, load average |
| Tasks | 전체 프로세스 수, 실행 중 / 대기 / 중지 / 좀비 프로세스 수 |
| %Cpu(s) | CPU 사용률 (사용자/시스템/유휴 등 비율) |
| Mem / Swap | 전체·사용·여유 메모리 및 스왑 용량 |
| 프로세스 목록 | PID, USER, PR, NI, VIRT, RES, SHR, S, %CPU, %MEM, TIME+, COMMAND |

### 주요 옵션
| 옵션 | 설명 |
|------|------|
| `-d <초>` | 화면 갱신 주기를 지정 |
| `-p <PID>` | 특정 PID의 프로세스만 모니터링 |
| `-u <사용자>` | 특정 사용자의 프로세스만 표시 |
| `-n <횟수>` | 지정한 횟수만큼 갱신 후 종료 |
| `-b` | 배치 모드(파일 리다이렉션 등에 활용) |

### 실행 중 단축키
- `P` : CPU 사용률 기준 정렬
- `M` : 메모리 사용률 기준 정렬
- `k` : 프로세스 종료(kill) 입력
- `q` : 종료

### 사용 예제
```bash
top              # 기본 실행
top -u ubuntu    # ubuntu 사용자의 프로세스만 보기
top -d 1         # 1초마다 갱신
```

## 2. `ps`

### 개요
`ps`는 **Process Status**의 줄임말로, 실행 중인 프로세스의 상태를 **현재 시점에 한 번** 출력하는 정적인 명령어입니다. `top`과 달리 실시간 갱신은 되지 않지만, 출력 결과를 `grep`, `awk` 등과 조합해 스크립트에서 활용하기 좋습니다.

### 기본 문법
```bash
ps [옵션]
```

### 주요 옵션
| 옵션 | 설명 |
|------|------|
| `-e` 또는 `-A` | 시스템의 모든 프로세스 표시 |
| `-f` | 전체(full) 형식으로 출력 (PPID, 시작 시간 등 포함) |
| `-u <사용자>` | 특정 사용자의 프로세스 출력 |
| `aux` | BSD 스타일로 모든 프로세스를 자세히 출력 (가장 많이 쓰임) |
| `-l` | 긴 형식(long format)으로 출력 |

### `ps aux` 출력 컬럼 설명
| 컬럼 | 의미 |
|------|------|
| USER | 프로세스 소유자 |
| PID | 프로세스 ID |
| %CPU | CPU 사용률 |
| %MEM | 메모리 사용률 |
| VSZ / RSS | 가상 메모리 / 실제 사용 메모리(KB) |
| STAT | 프로세스 상태 (`R` 실행, `S` 대기, `Z` 좀비, `T` 중지 등) |
| START | 프로세스 시작 시각 |
| TIME | 누적 CPU 사용 시간 |
| COMMAND | 실행 명령어 |

### 사용 예제
```bash
ps              # 현재 터미널의 프로세스만 출력
ps -ef          # 전체 프로세스를 full 형식으로 출력
ps aux | grep nginx   # nginx 관련 프로세스만 검색
```

## 3. `jobs`

### 개요
`jobs`는 **현재 로그인한 쉘(세션)** 안에서 백그라운드(`&`)로 실행했거나, `Ctrl+Z`로 일시 중지(stopped)시킨 작업 목록을 보여주는 명령어입니다. `ps`나 `top`과 달리 시스템 전체가 아닌 **해당 쉘 세션 안의 작업**만 보여준다는 점이 특징입니다.

### 기본 문법
```bash
jobs [옵션]
```

### 주요 옵션
| 옵션 | 설명 |
|------|------|
| `-l` | Job 번호와 함께 PID도 출력 |
| `-r` | 실행(running) 중인 작업만 출력 |
| `-s` | 중지(stopped)된 작업만 출력 |

### 작업 제어와 함께 사용하는 명령어
| 명령어 | 설명 |
|--------|------|
| `명령어 &` | 명령어를 백그라운드로 실행 |
| `Ctrl + Z` | 실행 중인 작업을 일시 중지하고 백그라운드로 보냄 |
| `fg %번호` | 지정한 job 번호를 포그라운드로 전환 |
| `bg %번호` | 지정한 job 번호를 백그라운드에서 계속 실행 |

### 사용 예제
```bash
sleep 100 &     # sleep 명령을 백그라운드로 실행
jobs            # [1]+  Running    sleep 100 &
jobs -l         # job 번호 + PID까지 함께 출력
fg %1           # 1번 작업을 다시 포그라운드로 가져오기
```

## 4. `kill`

### 개요
`kill`은 지정한 PID(또는 job 번호)에 **시그널(signal)**을 전달하여 프로세스를 종료하거나 제어하는 명령어입니다. 이름과 달리 단순히 "죽이는" 기능뿐 아니라 일시정지, 재시작 등 다양한 시그널을 보낼 수 있습니다.

### 기본 문법
```bash
kill [시그널] PID
kill [시그널] %job번호
```

### 주요 시그널
| 시그널 번호 | 시그널 이름 | 설명 |
|------------|-------------|------|
| 1 | `SIGHUP` | 연결 종료 시 / 설정 재읽기 신호 |
| 2 | `SIGINT` | `Ctrl + C`와 동일, 인터럽트(중단) 신호 |
| 9 | `SIGKILL` | 강제 종료 (즉시, 무조건 종료) |
| 15 | `SIGTERM` | 정상적인 종료 요청 (기본값, 프로세스가 처리 후 종료 가능) |
| 18 | `SIGCONT` | 중지된 프로세스를 다시 실행 |
| 19 | `SIGSTOP` | 프로세스를 일시 중지 |

### 관련 명령어
| 명령어 | 설명 |
|--------|------|
| `kill -9 PID` | 프로세스를 강제 종료 |
| `kill -l` | 사용 가능한 모든 시그널 목록 출력 |
| `killall <이름>` | 프로세스 이름으로 동일한 이름의 모든 프로세스 종료 |
| `pkill <패턴>` | 패턴과 일치하는 프로세스를 검색하여 종료 |

### 사용 예제
```bash
kill 1234          # PID 1234에 SIGTERM(15) 전송, 정상 종료 요청
kill -9 1234       # PID 1234를 강제 종료
kill -l            # 사용 가능한 시그널 목록 확인
kill %1            # 1번 job 종료
```
